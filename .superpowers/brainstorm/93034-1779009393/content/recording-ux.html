How does a row enter "recording" mode?

This dictates row layout. Same choice applies to both style flavors.

A

Click-to-record (inline)

Default row shows the binding as a label (⌘S or "No shortcut"). Click the label area → the cell becomes a ShortcutRecorderView in place. Capture finishes → cell reverts to label.

Compact · feels like Xcode's Key Bindings pane · one-click to edit

B

Always-on recorder

Every row shows a live ShortcutRecorderView always. Click anywhere on it to start recording; bare-click elsewhere finalizes (existing ShortcutField behavior). No mode toggle.

Simplest mental model · always shows ShortcutField field chrome · taller rows

C

Row-click → sheet / popover

Row is read-only display. Click row → a small sheet (or popover) opens with the recorder + reset + cancel buttons + room for a "currently bound to: ⌘S" hint and conflict messages.

More room for affordances · two-click workflow · matches System Settings → Keyboard → Shortcuts
